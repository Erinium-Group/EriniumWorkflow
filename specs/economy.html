Systeme Economique

Monnaie virtuelle, transactions joueurs, commandes admin et integrations avec la banque de faction et le cout de creation.

Termine EriData Singleton Dollar ($)

01 Vue d'ensemble

Le systeme economique d'EriniumFaction fournit une monnaie virtuelle unique par serveur, persistante par joueur, sans balance negative possible. Il sert de socle pour les transactions entre joueurs, la banque de faction et les couts de jeu configurables.

Parametre Valeur Notes
Monnaie Dollar ($) Symbole $ affiche apres le montant formate
Persistance EriData par joueur @Sync pour synchronisation client
Balance par defaut 0$ Configurable via EconomyData
Balance negative NON withdraw() retourne false si solde insuffisant
Format affichage %.2f$ Ex : 1 234.50$
Valeur max Double.MAX_VALUE Pas de cap hard-code cote joueur
ℹ
Principe fondamental — Toutes les operations monetaires passent exclusivement par EconomyManager. Aucun acces direct au champ balance de EconomyData depuis le reste du code.

02 Architecture technique

EconomyData.java

POJO annote @EriData, attache a chaque joueur, persiste entre les sessions et survit a la mort.

@EriData(modId = "eriniumfaction", scope = DataScope.PLAYER, persistOnDeath = true) public class EconomyData { @Sync public double balance = 0.0; }

EconomyManager.java

Singleton central. Fournit l'API complete pour toutes les operations monetaires.

double getBalance(EntityPlayer player)
Retourne le solde actuel du joueur.
void setBalance(EntityPlayer player, double amount)
Definit la balance a une valeur precise. Usage admin uniquement.
void deposit(EntityPlayer player, double amount)
Ajoute un montant au solde du joueur. Toujours positif.
boolean withdraw(EntityPlayer player, double amount)
Retire un montant. Retourne false si solde insuffisant, true si succes.
Verifie hasEnough() avant de modifier la balance.
boolean transfer(EntityPlayer from, EntityPlayer to, double amount)
Transfert atomique entre deux joueurs. Retourne false si l'expediteur n'a pas les fonds.
boolean hasEnough(EntityPlayer player, double amount)
Verifie si le joueur peut se permettre une depense sans effectuer l'operation.
String formatBalance(double amount)
Formate un montant en chaine lisible avec symbole $. Ex : 1 234.50$.
void reset(EntityPlayer player)
Remet la balance a 0. Usage admin uniquement.

Enregistrement

Dans CommonProxy.preInit() :

DataManager.init(); DataManager.register(EconomyData.class);
✓
Sync automatique — Le champ @Sync balance est automatiquement envoye au client a chaque DataManager.save(). Pas de packet manuel necessaire.

03 Commandes joueur

Commandes accessibles a tous les joueurs connectes. Aucune permission speciale requise (niveau 0).

Commande Description Permission Auto-completion
/money Voir son propre solde niveau 0 Aucun argument
/money <joueur> Voir le solde d'un autre joueur connecte niveau 0 PlayerArg — noms en ligne
/pay <joueur> <montant> Transfert de fonds vers un autre joueur niveau 0 PlayerArg + montant libre
/money pay <joueur> <montant> Alias de /pay niveau 0 PlayerArg + montant libre

Comportement /pay

  • Verifie que joueur != soi-meme — message d'erreur si auto-transfert
  • Verifie que le montant est strictement positif (> 0)
  • Appelle EconomyManager.transfer() — retourne false si fonds insuffisants
  • Message de confirmation envoye aux deux joueurs (expediteur et destinataire)
  • Le destinataire peut etre hors ligne — message envoye a sa prochaine connexion via EriChat

Messages de retour

// Solde propre "Votre solde : 1 234.50$" // Solde autre joueur "Solde de Steve : 567.00$" // Paiement reussi (expediteur) "Vous avez envoye 100.00$ a Steve. Nouveau solde : 1 134.50$" // Paiement recu (destinataire) "Alex vous a envoye 100.00$. Nouveau solde : 667.00$" // Erreur fonds insuffisants "Fonds insuffisants. Solde actuel : 50.00$"

04 Commandes admin

Commandes reservees aux joueurs ayant la permission economy.admin. La verification passe par RankManager.hasPermission(), pas par le niveau OP Minecraft.

Commande Description Permission Operation
/money give <joueur> <montant> Ajouter de l'argent a un joueur economy.admin EconomyManager.deposit()
/money set <joueur> <montant> Definir la balance exacte d'un joueur economy.admin EconomyManager.setBalance()
/money remove <joueur> <montant> Retirer de l'argent a un joueur economy.admin EconomyManager.withdraw()
/money reset <joueur> Remettre la balance a zero economy.admin EconomyManager.reset()
⚠
Systeme de permissions — La permission economy.admin doit etre ajoutee dans le grade admin via /rank addperm admin economy.admin. Elle supporte le wildcard economy.* pour couvrir toutes les commandes economie.

Logs admin

Chaque operation admin est loguee dans le fichier serveur avec le format :

[EriniumFaction] [ECONOMY] AdminPlayer a give 500.00$ a TargetPlayer (nouveau solde : 1500.00$)

05 Integration banque de faction

La banque de faction est un compte collectif lie a FactionData.bank. Les deposits et retraits font le lien entre le compte perso du joueur et la banque commune.

Depot (joueur → banque)

  • Verifie que la banque est activee (FactionConfig.factionBankEnabled)
  • Verifie que le montant est positif
  • Verifie que le solde joueur est suffisant via EconomyManager.hasEnough()
  • Verifie que la banque ne depasse pas FactionConfig.factionBankMax
  • Effectue EconomyManager.withdraw(player, montant)
  • Ajoute le montant a factionInfo.bank et sauvegarde
  • Notifie le joueur et le chat de faction

Retrait (banque → joueur)

  • Verifie les permissions de retrait selon le role faction (leader / officier)
  • Verifie que le solde banque est suffisant
  • Retire le montant de factionInfo.bank
  • Effectue EconomyManager.deposit(player, montant)
  • Notifie le joueur et le chat de faction
Erreur possible Message affiche
Banque desactivee "La banque de faction est desactivee sur ce serveur."
Solde joueur insuffisant (depot) "Fonds insuffisants. Solde actuel : X$"
Banque pleine (depot) "La banque est pleine (max : X$)."
Solde banque insuffisant (retrait) "La banque ne contient pas assez de fonds. Solde banque : X$"
Permission insuffisante (retrait) "Vous n'avez pas la permission de retirer de la banque."

06 Integration cout de creation de faction

Si FactionConfig.factionCreationCost est superieur a 0, la creation d'une faction necessite un paiement prealable du fondateur.

Flux de creation

  • Le joueur execute /f create <nom>
  • FactionManager verifie le cout : si cost > 0, appelle EconomyManager.hasEnough(player, cost)
  • Si fonds insuffisants → message d'erreur avec le cout requis, annulation
  • Si fonds suffisants → EconomyManager.withdraw(player, cost) puis creation de la faction
  • Message de confirmation indiquant le montant debite

Cle lang dynamique

// Message si cout insuffisant "Creer une faction coute %.0f$. Votre solde : %.0f$" // Message apres creation reussie avec cout "Faction creee ! %.0f$ ont ete debites de votre compte."
✓
Cout = 0 par defaut — Avec la config a 0, aucune verification economique n'est effectuee. Le comportement est identique a l'ancien systeme sans economie.

07 Configuration

Entrees FactionConfig existantes liees au systeme economique.

Cle config Type Defaut Description
factionCreationCost double 0 Cout en $ pour creer une faction. 0 = gratuit.
factionBankEnabled boolean true Activer ou desactiver la banque de faction.
factionBankMax double 1000000 Balance maximale de la banque de faction.
ℹ
factionCreationCost — Valeur min : 0, valeur max recommandee : 1 000 000. Au-dela, risque de bloquer indefinitivement les joueurs sans economie etablie sur le serveur.
