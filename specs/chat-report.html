Systeme de Report Chat

Signalement de messages inappropriés directement depuis le chat via une icône cliquable. Outils de consultation et d'action pour les modérateurs. Auto-mute configurable en cas de signalements répétés.

Moderation Chat Integration EriChat Anti-Spam Planifie

01Contexte & Objectifs

Pourquoi ce systeme

Sur un serveur de 500-1000 joueurs, le chat est actif et les modérateurs ne peuvent pas tout lire. Le système de report permet aux joueurs de signaler un message en un clic, sans avoir à mémoriser une commande ou copier un texte. Les modérateurs reçoivent les reports en temps réel et peuvent agir directement depuis le chat.

Objectifs

  • Friction minimale — un clic sur l'icône suffit pour signaler, pas besoin de copier le message
  • Icône invisible pour l'auteur — le joueur qui envoie un message ne voit pas son propre icône de report
  • Cooldown anti-spam — un joueur ne peut pas flood les reports (1 report par minute)
  • Buffer consultable — les 50 derniers reports sont accessibles aux mods via /reports
  • Auto-mute — si un joueur accumule trop de reports en peu de temps, il est muté automatiquement
  • Intégration ChatManager — s'appuie sur le système de chat existant, pas de mixin fragile
ℹ
Dependance : Ce système requiert que le système d'administration (administration.html) soit implémenté en premier, car l'auto-mute appelle AdminManager.applyMute() pour enregistrer la sanction au casier judiciaire.

02Icone Cliquable dans le Chat

Apercu visuel

Chaque message dans le chat est précédé d'une icône rouge cliquable. Voici comment un message apparaît pour les autres joueurs :

⚠ [Guerriers] JoueurXYZ : Bonjour à tous !

L'auteur du message voit son propre message sans l'icône :

[Guerriers] JoueurXYZ : Bonjour à tous !

Construction du ITextComponent

// Construction de l'icône de report — ReportManager.buildReportIcon() public static ITextComponent buildReportIcon(String targetName, int messageId) { TextComponentString icon = new TextComponentString("⚠ "); Style style = new Style(); style.setColor(TextFormatting.RED); style.setBold(true); // Clic → exécute /report <pseudo> Message+inappropriate style.setClickEvent(new ClickEvent( ClickEvent.Action.RUN_COMMAND, "/report " + targetName + " Message inapproprié" )); // Hover → texte d'aide style.setHoverEvent(new HoverEvent( HoverEvent.Action.SHOW_TEXT, new TextComponentString("§7Cliquer pour signaler ce message\n§8ID: #" + messageId) )); icon.setStyle(style); return icon; }

ID de message

Chaque message reçoit un identifiant entier incrémental (messageIdCounter dans ReportManager, thread-safe via AtomicInteger). L'ID est visible dans le hover de l'icône et permet d'identifier quel message exact a été signalé lors de la consultation du buffer.

Distribution selective

ServerChatEvent
Le message est intercepté avant distribution. ReportManager.onChat() est appelé.
Attribution d'un ID
Un ID unique est assigné via messageIdCounter.getAndIncrement(). Le message est mis en buffer (50 derniers).
Construction du message complet
Le composant final est : [icône] + message original. L'icône est un ITextComponent séparé prepend au message existant.
Distribution
L'event est cancelled. Le message avec icône est envoyé à tous les joueurs SAUF l'auteur. L'auteur reçoit le message sans icône (message original).

03Structure de Donnees

ReportEntry — un report

public class ReportEntry { public final int messageId; // ID unique du message signalé public final String reporterName; // Pseudo du signaleur public final UUID reporterUUID; public final String targetName; // Pseudo du signalé public final UUID targetUUID; public final String message; // Contenu du message signalé (tronqué à 256 chars) public final String reason; // Raison choisie par le signaleur public final long timestamp; // System.currentTimeMillis() public boolean dismissed; // Ignoré par un mod via /reports }

Buffer de messages recents — MessageEntry

public class MessageEntry { public final int id; public final String authorName; public final UUID authorUUID; public final String content; // Message brut (sans formatage chat) public final long timestamp; }

Stockage en memoire

public class ReportManager { // Buffer circulaire des 50 derniers reports private final ArrayDeque<ReportEntry> reportsBuffer = new ArrayDeque<>(50); // Buffer circulaire des 100 derniers messages (pour associer un report à son message) private final ArrayDeque<MessageEntry> messagesBuffer = new ArrayDeque<>(100); // Cooldown : UUID → timestamp du dernier report envoyé private final Map<UUID, Long> lastReportTime = new HashMap<>(); // Reports par cible dans la fenêtre glissante (UUID → liste de timestamps) private final Map<UUID, List<Long>> reportsAgainst = new HashMap<>(); // Compteur de messages (AtomicInteger pour thread-safety) private final AtomicInteger messageIdCounter = new AtomicInteger(1); }
⚠
Stockage volatile : Le buffer de reports est uniquement en mémoire et ne survit pas aux redémarrages. Il représente les signalements actifs de la session en cours. Pour un historique long terme, les reports déclenchant une sanction sont enregistrés dans le casier judiciaire via AdminManager.

04Commande /report

Syntaxe

/report <joueur> [raison]

Comportement

  • Permission : aucune — tout le monde peut signaler
  • Cooldown : 1 report par minute par joueur (configurable). Message si trop tôt : "Vous avez déjà signalé un joueur récemment. Attendez {N} secondes."
  • Auto-raison : si la raison est omise, elle vaut "Message inapproprié"
  • Confirmation au signaleur : "[REPORT] Votre signalement a été transmis aux modérateurs." (message discret, en gris)
  • Alerte aux mods : tous les joueurs en ligne avec la permission eriniumfaction.mod.lookup reçoivent l'alerte en rouge
  • Enregistrement : le report est ajouté au buffer avec timestamp, recherche du dernier message de la cible dans messagesBuffer
  • Pas de report de soi-même : vérification sender == target

Alerte aux moderateurs

// Message aux mods (via EriChat pour le formatage riche) EriChat.create() .text("[REPORT] ").color(TextFormatting.RED).bold() .text(reporterName).color(TextFormatting.YELLOW) .text(" a signalé ").color(TextFormatting.GRAY) .text(targetName).color(TextFormatting.WHITE).bold() .text("\nDernier message : \"").color(TextFormatting.GRAY) .text(lastMessage).color(TextFormatting.WHITE) // tronqué à 80 chars .text("\"\nRaison : ").color(TextFormatting.GRAY) .text(reason).color(TextFormatting.GOLD) .text(" | ").color(TextFormatting.DARK_GRAY) .text("[Mute]").color(TextFormatting.RED).bold() .command("/mute " + targetName + " 30m Report joueur") .hover("Muter " + targetName + " 30 minutes") .text(" ") .text("[Warn]").color(TextFormatting.GOLD).bold() .command("/warn " + targetName + " Report joueur") .hover("Avertir " + targetName) .toPlayersWithPermission("eriniumfaction.mod.lookup");

Gestion du cooldown

// Vérification du cooldown — O(1) long now = System.currentTimeMillis(); long lastTime = lastReportTime.getOrDefault(player.getUniqueID(), 0L); long cooldownMs = config.reportCooldownSeconds * 1000L; if (now - lastTime < cooldownMs) { long remaining = (cooldownMs - (now - lastTime)) / 1000; // Envoyer message d'erreur au joueur return; } lastReportTime.put(player.getUniqueID(), now);

05Commande /reports

Syntaxe

/reports [page]

Comportement

  • Permission : eriniumfaction.mod.reports
  • Affiche les reports du buffer, du plus récent au plus ancien
  • 8 reports par page, pagination via ChatPaginator
  • En-tête : "=== Reports récents ({N} signalements) ==="
  • Chaque entrée : [heure] Signaleur a signalé Cible — "message tronqué" — Raison
  • Actions cliquables sur chaque ligne : [Mute 30m] [Warn] [Ignorer]
  • Les reports ignorés (dismissed=true) sont affichés en grisé
  • Si aucun report : "Aucun report récent."

Affichage avec ChatPaginator

// En-tête EriChat.create() .text("═══ Reports récents (" + count + ") ═══").color(TextFormatting.DARK_RED).bold() .to(sender); // Par report dans la pagination for (ReportEntry r : reports) { EriChat.create() .text("[" + formatHour(r.timestamp) + "] ").color(TextFormatting.DARK_GRAY) .text(r.reporterName).color(TextFormatting.YELLOW) .text(" → ").color(TextFormatting.GRAY) .text(r.targetName).color(TextFormatting.WHITE).bold() .text(" — \"" + truncate(r.message, 50) + "\"").color(TextFormatting.GRAY) .text(" ") .text("[M]").color(TextFormatting.RED).command("/mute " + r.targetName + " 30m Report").hover("Muter 30min") .text(" ") .text("[W]").color(TextFormatting.GOLD).command("/warn " + r.targetName + " Report joueur").hover("Avertir") .text(" ") .text("[✓]").color(TextFormatting.GREEN).command("/reports dismiss " + r.messageId).hover("Ignorer ce report") .to(sender); }

Sous-commande dismiss

/reports dismiss <messageId> — marque un report comme ignoré (dismissed=true)

06Auto-Mute par Reports

Principe

Si un joueur reçoit trop de signalements dans une fenêtre de temps courte, il est muté automatiquement. Cette fonctionnalité est configurable et peut être désactivée.

ParametreDefautDescription
autoMuteAfterReports5Nombre de reports uniques dans la fenêtre pour déclencher l'auto-mute
autoMuteReportWindow10 minutesFenêtre glissante en minutes
autoMuteDuration15 minutesDurée du mute automatique déclenché par les reports
autoMuteEnabledtrueActiver ou désactiver l'auto-mute par reports

Implementation

// Appelé dans ReportManager.addReport() après chaque nouveau report private void checkAutoMute(UUID targetUUID, String targetName) { if (!config.autoMuteEnabled) return; List<Long> times = reportsAgainst.computeIfAbsent(targetUUID, k -> new ArrayList<>()); long now = System.currentTimeMillis(); long windowMs = config.autoMuteReportWindowMinutes * 60_000L; // Purger les timestamps hors fenêtre times.removeIf(t -> now - t > windowMs); times.add(now); if (times.size() >= config.autoMuteAfterReports) { // Déclencher le mute via AdminManager AdminManager.getInstance().applyMute( targetUUID, targetName, config.autoMuteDurationSeconds, "[AUTO] Signalements multiples (" + times.size() + " reports en " + config.autoMuteReportWindowMinutes + "min)", "SYSTEME" ); times.clear(); // Reset pour éviter le spam de mutes // Broadcast aux mods AdminManager.getInstance().broadcastToStaff( "[AUTO-REPORT] " + targetName + " muté automatiquement (" + times.size() + " reports)" ); } }

Reports uniques uniquement

Pour éviter qu'un seul joueur déclenche l'auto-mute en envoyant 5 reports, le système compte les reporters uniques : si JoueurA reporte 5 fois la même personne, cela ne compte que pour 1. Il faut 5 reporters différents pour déclencher l'auto-mute.

07Integration ChatManager

Point d'integration

L'injection de l'icône se fait dans le handler de l'événement ServerChatEvent. C'est le point le plus propre : l'event est cancellable, les ITextComponent sont accessibles, et la distribution manuelle aux joueurs permet de différencier l'auteur des autres.

@SubscribeEvent public void onServerChat(ServerChatEvent event) { EntityPlayerMP author = event.getPlayer(); ITextComponent originalMsg = event.getComponent(); // Vérification mute (AdminManager) if (AdminManager.getInstance().isMuted(author)) { // ... envoyer message "vous êtes muet" event.setCanceled(true); return; } // Enregistrer le message dans le buffer int msgId = ReportManager.getInstance().recordMessage(author, originalMsg.getUnformattedText()); // Construire la version "avec icône" TextComponentString reportIcon = ReportManager.buildReportIcon(author.getName(), msgId); ITextComponent withIcon = reportIcon.appendSibling(originalMsg); // Distribution manuelle — cancel l'event Forge normal event.setCanceled(true); MinecraftServer server = author.getServer(); for (EntityPlayerMP player : server.getPlayerList().getPlayers()) { if (player.getUniqueID().equals(author.getUniqueID())) { // L'auteur reçoit le message sans icône player.sendMessage(originalMsg); } else { // Les autres reçoivent le message avec icône player.sendMessage(withIcon); } } }
✓
Compatible avec le chat faction : Si le ChatManager gère déjà différents canaux (faction, allié, global), le hook de report s'applique sur les messages du canal global uniquement. Les messages privés (/msg) n'ont pas d'icône de report.

08Architecture

ReportManager
Singleton principal. Buffer des reports et des messages. Cooldown tracking. Auto-mute logic. Construction des ITextComponent d'icône.
ReportEntry
POJO immuable. Représente un signalement : reporter, cible, message, raison, timestamp, dismissed.
MessageEntry
POJO immuable. Représente un message chat avec son ID, son auteur et son contenu brut. Buffer de 100 messages max.
ReportCommand
Gère /report (public) et /reports (mod). Sous-commande dismiss. Auto-complétion sur les pseudos.
ReportEventHandler
@Mod.EventBusSubscriber. Hook ServerChatEvent pour l'injection de l'icône et la distribution sélective.
ReportConfig
Configuration : cooldown, buffer size, seuils auto-mute, durée auto-mute.
