Enchantements d'Erinium

Quarante enchantements custom accessibles uniquement via des bibliotheques moddees placees autour de la table d'enchantement — s'ajoutent aux enchantements vanilla sans les remplacer.

01 Contexte & Objectifs

Philosophie centrale

EriniumFaction n'ecrase pas le systeme d'enchantement vanilla. Il l'etend. Les enchantements Erinium s'ajoutent au pool normal de la table d'enchantement et deviennent disponibles uniquement a partir du niveau 50, c'est-a-dire uniquement lorsque des bibliotheques moddees sont presentes autour de la table. Avec des bibliotheques vanilla seules (max niveau 30), aucun enchantement custom n'apparait jamais.

Ce choix preserve la courbe d'apprentissage vanilla pour les nouveaux joueurs tout en offrant une couche de profondeur considerable pour ceux qui investissent dans les materiaux de haut tier. Les bibliotheques moddees sont les cles d'acces a ce contenu.

Objectifs

  • Complementarite — Chaque enchantement custom comble un manque du vanilla ou amplifie une mecanique existante de facon interessante en PvP 1.8 (pas de cooldown).
  • Progression materielle — Les bibliotheques sont crafted avec les minerais Valterite, Aetherite et Erinium du systeme Combat & Equipement. Enchantemment haut niveau = investissement en ressources haut tier.
  • Lisibilite — Les effets doivent rester comprehensibles pour le joueur cible. Pas d'effets invisibles ou imperceptibles en combat rapide.
  • Equilibrage PvP 1.8 — Le serveur utilise un mod PvP 1.8 (pas de cooldown d'attaque, jitter click). Tous les enchantements d'armes sont equilibres pour ce contexte : pas de bonus d'attack speed (attribut inutile), focus sur les degats, le knockback, la mobilite et la survie.
  • Integration Forge propre — Extension du registre d'enchantements Forge standard via RegistryEvent.Register<Enchantment>. Les Mixin modifient la logique de la table sans toucher au code vanilla en dehors des points d'injection.
⚠
PvP 1.8 — regle absolue : L'attribut ATTACK_SPEED est inutile sur ce serveur. Aucun enchantement ne doit baser son effet sur la vitesse d'attaque. Les enchantements d'armes se concentrent sur les degats bruts, le knockback, les effets de statut et les mecaniques de jeu actif (combos, stacks de hits, sprinting).

Systeme de puissance bookshelf

La table d'enchantement vanilla lit le nombre de bibliotheques dans un rayon de 2 blocs et capefie a 15 (donne le niveau 30 max). Les bibliotheques moddees sont detectees via Mixin et comptent plusieurs fois dans ce calcul :

BlocPuissance equivalentePuissance requise pour palierConfiguration exemple
Bibliotheque vanilla1—15 vanilla = niveau 30
Bibliotheque Valterite3~17 total pour niv. 505 Valterite + 2 vanilla = 17
Bibliotheque Aetherite5~23 total pour niv. 704 Aetherite + 3 vanilla = 23
Bibliotheque Erinium7~30 total pour niv. 904 Erinium + 2 vanilla = 30
ℹ
Formule de conversion niveau : niveau_max = min(90, puissance_totale * 3). Une puissance de 17 donne niveau 51, arrondie a 50. Les bibliotheques moddees peuvent etre melangees entre elles et avec des bibliotheques vanilla.

Regles de placement

Les regles de placement sont identiques aux bibliotheques vanilla :

  • Placees a exactement 1 bloc de distance de la table d'enchantement (laisser un espace d'air entre la bibliotheque et la table)
  • Meme hauteur (Y) que la table ou 1 bloc au-dessus
  • Pas de bloc opaque entre la bibliotheque et la table (un bloc d'air obligatoire)
  • Positionnees dans un rayon de 5x5 blocs autour de la table
  • Peuvent etre mixees : bibliotheques vanilla + bibliotheques moddees dans le meme arrangement

Recettes de craft

Les recettes sont en shaped 3x3. Les lingots correspondent aux materiaux du systeme Combat & Equipement.

03 Table d'Enchantement Modifiee

Depassement du niveau 30

La table d'enchantement vanilla est limitee au niveau 30. Trois Mixin liftent cette limite et modifient le calcul de la puissance de bibliotheque pour accepter les blocs moddees. La GUI de la table est mise a jour via Mixin pour afficher les niveaux 50/70/90.

Minimum enchantements garantis

Le minimum garanti signifie que l'item recevra au moins N enchantements au total (vanilla + custom confondus) lors de l'enchantement. Ce n'est pas un minimum de custom uniquement — c'est la somme totale. Le systeme reroll en interne la selection jusqu'a satisfaire cette contrainte, avec un maximum de 5 tentatives pour eviter les boucles infinies.

✓
Pourquoi ce minimum garanti ? Investir dans des bibliotheques Erinium doit se ressentir immediatement. Un item enchante niveau 90 doit etre spectaculairement different d'un item niveau 30. Le minimum garanti de 9 enchantements avec des bibliotheques Erinium assure cet effet "wow" a chaque utilisation.

Points d'injection Mixin

  • MixinContainerEnchantment — override de onCraftMatrixChanged() pour recalculer la puissance bookshelf en detectant les blocs moddees. Override de getEnchantmentLevel() pour lever le plafond a 90.
  • MixinEnchantmentHelper — override de buildEnchantmentList() pour inclure les enchantements custom dans le pool de selection et appliquer le minimum garanti.
  • MixinBlockBookshelf — ajout d'une interface IBookshelfPower ; les blocs moddees implementent cette interface. La detection generique remplace le check de type hardcode de vanilla.
⚠
Compatibilite production : Les Mixin sont charges via MixinBooter (IEarlyMixinLoader) avant les classes vanilla. En production, MixinBooter doit etre dans le dossier mods. Le refmap obfusque est genere a la compilation via l'annotation processor de MixinGradle.

05 Enchantements d'Armes

Dix enchantements pour epees et haches. Tous respectent le contexte PvP 1.8 (pas de cooldown d'attaque). Les mecaniques se concentrent sur les combos de clics rapides, le positionnement, le knockback et la gestion de la survie.

Fracture
Max III Niv. 35+
Les coups critiques (saut + attaque) creeent une onde de choc AoE au point d'impact.
I — 20% dmg, 2 blocs II — 30% dmg, 3 blocs III — 40% dmg, 4 blocs
Le pourcentage est calcule sur les degats du coup critique. Pas de degats aux blocs. L'AoE ignore les entites non-joueurs a moins de 1 bloc (protection contre les mobs proches).
Siphon
Max III Niv. 50+
Lifesteal : une portion des degats infligees est rendue en PV au porteur.
I — 5% vol II — 8% vol III — 12% vol
Applique uniquement sur les degats directs (pas sur les effets de statut du a l'arme). Le soin est en demi-coeurs arrondis au superieur. Ne fonctionne pas si le porteur est a plein PV.
Ancrage
Max II Niv. 40+
Modifie le ratio knockback : reduit celui recu, augmente celui inflige. Crucial en PvP 1.8 ou le knockback est la mecanique principale.
I — -20% recu, +15% inflige II — -35% recu, +25% inflige
Se cumule avec l'enchantement Knockback vanilla (qui, lui, augmente uniquement le knockback inflige). Incompatible avec Onde de Choc II (les deux enchantements se chevauchent sur la mecanique de push).
Faille
Max III Niv. 50+
Les frappes consecutives sur la meme cible ignorent de plus en plus son armure. Le compteur se remet a zero si le porteur change de cible.
I — +3%/frappe (max 15%) II — +5%/frappe (max 25%) III — +7%/frappe (max 35%)
Le compteur de cible est stocke dans une Map temporaire (non persistante). Reset si aucune frappe pendant 3 secondes. Mecanique forte en PvP 1.8 ou les combos de clics rapides sont la norme.
Resonance
Max II Niv. 45+
Les frappes marquent l'ennemi. Les allies du porteur (membres de la meme faction) infligent des degats bonus aux cibles marquees.
I — +8% pour 3s II — +12% pour 5s
Le marquage est visible via une particule violette sur la cible. Un seul marqueur actif par cible a la fois (le dernier applique ecrase le precedent). Oriente pour le jeu en equipe faction.
Volatilite
Max III Niv. 55+
Chaque N-ieme frappe est amplifiee. Le compteur est par-arme, stocke en NBT.
I — 6e frappe +30% II — 5e frappe +40% III — 4e frappe +50%
La frappe amplifiee est signalee par une particule d'explosion dorée. Le compteur NE se remet PAS a zero au changement de cible (mecanique d'anticipation, pas de combo). Compte uniquement les frappes qui atterrissent.
Cristallisation
Max I Niv. 70+
Apres 3 kills sans mourir, le porteur gagne un bouclier absorbant 4 coeurs (8 PV d'absorption). Un seul niveau par sa puissance.
I — 3 kills = 4 coeurs d'absorption
Le bouclier est un effet Absorption (MobEffects.ABSORPTION). Le compteur de kills est stocke dans la capability joueur et resete a la mort. Le bouclier ne se cumule pas (si deja actif, le compteur repart de 0).
Hemorragie
Max III Niv. 40+
Les frappes appliquent un saignement (DoT). Le saignement ne se cumule pas : si la cible est deja en train de saigner, la duree est rafraichie (pas d'empilement de dommages).
I — 0.5 coeur/2s pendant 4s II — 0.5 coeur/1.5s pendant 6s III — 1 coeur/2s pendant 6s
Implemente via un effet de statut custom (pas Wither, qui a un son different) applique par EriScheduler.
Onde de Choc
Max II Niv. 60+
Les frappes effectuees en sprint liberent une onde de choc en cone vers l'avant qui repousse les entites proches.
I — 3 blocs push, cone 4 blocs II — 5 blocs push, cone 6 blocs
L'onde de choc ne s'applique qu'a la cible principale si elle est dans le cone. Les entites dans le cone mais non frappees recoivent 50% de la force de push. Incompatible avec Ancrage II.
Voracite
Max II Niv. 50+
Les kills restaurent de la faim et de la saturation au porteur. Utile pour soutenir les combats prolonges.
I — +2 barres de faim/kill II — +4 barres + 1 saturation/kill
Uniquement sur kill de joueur (pas de mob). Ecouteur sur LivingDeathEvent filtre sur EntityPlayerMP.

06 Enchantements d'Armure

Dix enchantements pouvant etre places sur n'importe quelle piece d'armure. Certains effets s'empilent a travers les pieces (avec caps de securite), d'autres n'entrent en jeu que sur une piece specifique par logique de gameplay.

Fortification
Max III Niv. 35+
Reduit les degats recus lorsque le porteur reste immobile pendant au moins 1.5 secondes. Penalise le camping sans le rendre indestructible.
I — -8% II — -15% III — -22%
La detection d'immobilite compare la position actuelle a celle il y a 30 ticks. Si delta < 0.01 blocs, consideree immobile. L'effet se desactive immediatement au premier mouvement.
Deflexion
Max III Niv. 40+
Chance de defleter les projectiles (fleches, boules de feu) sans les subir. La fleche disparait sans infliquer de degats.
I — 8% II — 15% III — 22%
Cap de 40% si plusieurs pieces ont Deflexion (cumul additif plafonné). Implemente via Mixin sur EntityArrow.onUpdate() ou via ArrowHitEvent (Forge).
Resilience
Max II Niv. 50+
Lorsque les PV tombent sous 25%, le porteur gagne temporairement une reduction de degats supplementaire avec cooldown.
I — +15% pour 3s (CD 40s) II — +25% pour 4s (CD 30s)
L'activation est passive (auto-trigger). Ne se cumule pas avec une seconde piece ayant Resilience (un seul trigger actif a la fois par joueur).
Dispersion
Max II Niv. 45+
Chance d'aveugler l'attaquant lorsque le porteur est frappe au corps a corps.
I — 8% chance, 1s blind II — 12% chance, 1.5s blind
Uniquement declenchee sur frappes au corps a corps (pas sur projectiles). Applique l'effet Blindness (MobEffects.BLINDNESS) sur l'attaquant.
Echo Defensif
Max I Niv. 55+
Lorsque le porteur est frappe 3 fois en 2 secondes par le meme joueur, il libere une impulsion de knockback radiale de 6 blocs. Cooldown de 15 secondes.
I — 3 frappes / 2s → push 6 blocs (CD 15s)
Anti-spam de clics. Tres efficace en PvP 1.8. Le compteur de frappes est par attaquant, stocke en cache joueur avec timestamp.
Regeneration Cristalline
Max III Niv. 50+
Regenere des PV lorsque le porteur est dans le territoire de sa faction (chunk revendique).
I — +0.25 coeur/5s II — +0.5 coeur/5s III — +0.5 coeur/3s
S'empile a travers les pieces d'armure, cap a +2 coeurs/5s total. Desactive pendant le combat (si un degat a ete recu dans les 5 dernieres secondes).
Carapace
Max III Niv. 35+
Reduit les degats d'explosion et d'AoE (TNT, creeper, grenades). Ne reduit pas les degats direct en melee.
I — -15% II — -25% III — -35%
Detecte via DamageSource.isExplosion(). S'additionne avec la Protection vanilla mais cap total a -60% de reduction d'explosion.
Absorption d'Impact
Max II Niv. 60+
Chance de nier completement le knockback d'une frappe recue (les degats sont toujours subis, seul le recul est annule).
I — 10% II — 20%
Tres puissant en PvP 1.8 ou le knockback est la mecanique centrale. Incompatible avec Carapace (deux enchantements defensifs majeurs, la combinaison serait trop forte).
Vitalite
Max III Niv. 45+
Augmente les PV maximum du porteur. S'empile a travers les pieces mais avec un cap global.
I — +1 coeur II — +2 coeurs III — +3 coeurs
Cap global de +6 coeurs toutes pieces confondues (via attribut MAX_HEALTH). Implemente via EntityPlayer.getAttributeMap().getAttributeInstance(SharedMonsterAttributes.MAX_HEALTH).
Represailles
Max II Niv. 70+
Lorsque le porteur subit des degats, une portion est refletee sur l'attaquant. Fonctionne sur tous types de degats (melee, projectiles, explosions). Meilleur que Thorns.
I — 10% reflet II — 18% reflet
Le reflet NE declenche PAS le cooldown de degats de l'attaquant. Les degats refletes sont de type DamageSource.MAGIC (neutres). Incompatible avec Thorns vanilla (redondance).

07 Enchantements d'Arc

Dix enchantements pour l'arc. Le serveur PvP 1.8 favorise les arcs a charge rapide et les combats a distance pour couvrir les allies. Les enchantements d'arc sont equilibres autour de cet usage.

Fleche Tracante
Max II Niv. 50+
Les fleches qui touchent une cible lui appliquent l'effet Glowing (lueur visible a travers les blocs), facilitant le suivi en combat.
I — 3s glow II — 6s glow
Glowing est implemente via Mixin sur EntityPlayer.getFlag() pour Minecraft 1.12.2 (l'effet n'existe pas nativement). La lueur est envoyee au client via packet custom.
Tir Perforant
Max III Niv. 45+
Les fleches traversent les entites sans s'arreter, permettant d'atteindre plusieurs joueurs en ligne.
I — traverse 1 entite II — traverse 2 entites III — traverse 3 entites
Le nombre de traversees restantes est stocke dans le NBT de l'entite fleche. Chaque traversee reduit les degats de 15%. Implemente via Mixin sur EntityArrow.onImpact().
Fleche Explosive
Max II Niv. 60+
Les fleches creent une petite explosion a l'impact. Aucun degat aux blocs.
I — 1.5 blocs, 30% degats fleche en AoE II — 2.5 blocs, 50% degats fleche en AoE
L'explosion est simulee via World.createExplosion() avec le flag de degats aux blocs desactive. Incompatible avec Tir Perforant (une fleche ne peut pas percer et exploser).
Tir Rapide
Max III Niv. 35+
Reduit le temps de charge de l'arc. En PvP 1.8, tirer plus vite est un avantage tactique majeur.
I — -15% charge II — -25% charge III — -35% charge
La charge minimale pour une fleche pleine puissance reste de 1 tick (pas de tir instantane). Implemente via Mixin sur ItemBow.onPlayerStoppedUsing().
Fleche Givrante
Max II Niv. 40+
Les fleches appliquent un ralentissement a l'impact, perturbant les combattants au corps a corps.
I — Slowness I pendant 2s II — Slowness II pendant 3s
Compatible avec la Fleche Tracante (combo utile : marquer et ralentir). Incompatible avec Fleche Explosive (les effets de statut ne s'appliquent pas sur les degats AoE).
Volee
Max II Niv. 70+
Le tir libere des fleches supplementaires en dispersion. Les fleches supplementaires font moins de degats.
I — +1 fleche (50% degats) II — +2 fleches (50% degats, dispersion elargie)
Les fleches supplementaires consomment des fleches en inventaire (1 fleche extra = 1 fleche en plus consommee). La dispersion est de +-8° I, +-12° II. Incompatible avec Infinity (trop fort avec fleches infinies).
Precision
Max III Niv. 50+
Les fleches sont moins affectees par la gravite, volant de facon plus rectiligne sur longue distance.
I — -20% gravite II — -40% gravite III — -60% gravite
Implemente via Mixin sur EntityArrow.onUpdate() en reduisant le vecteur gravitationnel. Les fleches restent affectees par la gravite a III (pas de trajectoire completement plate).
Fleche Vampirique
Max II Niv. 55+
Une portion des degats infligees par les fleches soigne le tireur.
I — 15% soin II — 25% soin
Fonctionne uniquement sur les degats directs de fleche (pas sur les fleches secondaires de Volee). Incompatible avec Siphon (meme mecanique sur des types d'armes differents — si le joueur porte les deux types d'armes avec ces enchants, c'est acceptable, mais pas sur le meme item).
Recul d'Arc
Max II Niv. 50+
Les fleches repoussent plus fort les cibles touchees. Excellent pour maintenir la distance en combat a distance.
I — +50% knockback II — +100% knockback
Se cumule avec l'enchantement Punch vanilla. Le knockback d'arc est horizontal comme en vanilla. Compatible avec Tir Perforant (chaque entite traversee recoit le knockback).
Homing
Max I Niv. 80+
Les fleches se courbent legerement vers l'entite la plus proche dans un rayon de 3 blocs autour de leur trajectoire. Guidage subtil, pas un aimbot.
I — correction douce, 3 blocs de detection
La correction est limitee a 3° par tick. Si aucune entite n'est dans le rayon, la fleche suit sa trajectoire normale. Le guidage ignore les entites alliees (meme faction). Necessite le niveau le plus eleve pour le debloquer (Niv. 80+).

08 Enchantements d'Outils

Dix enchantements pour les outils (pioche, hache, pelle, houe). Certains sont specifiques a un type d'outil (notes dans la description). Ces enchantements transforment profondement l'efficacite du minage et de l'agriculture en faction.

Excavation
Max III Niv. 40+
Mine une zone elargie en un seul clic. S'applique sur pioche, pelle et hache.
I — zone 1x2 II — zone 1x3 III — zone 3x3
Les blocs supplementaires minent uniquement si le bloc cible est mineable par l'outil. Ne mine pas les blocs hors de portee du joueur. Implemente via PlayerInteractEvent.LeftClickBlock avec un flag anti-recursion.
Auto-Smelt
Max I Niv. 35+
Les blocs mines droppent directement leur forme fondue : minerai de fer → lingot, minerai d'or → lingot d'or, sable → verre, etc.
I — drop forme fondue
Compatible avec Fortune : quand les deux sont presents, Fortune s'applique avec un rendement reduit a 50% (Fortune III donne normalement 1-4 drops → avec Auto-Smelt donne 1-2 lingots au lieu de 1-4 minerais bruts). Le joueur gagne en confort mais perd en quantite max vs Fortune seul. Compatible avec Silk Touch : si les deux sont sur le meme outil, Silk Touch a priorite.
Magnetisme
Max II Niv. 50+
Les drops de blocs sont automatiquement attires vers le joueur lors du minage.
I — 3 blocs de portee II — 6 blocs de portee
Uniquement lors du minage (pas en permanence pour les items au sol). Implemente en appliquant une velocity vers le joueur sur les EntityItem crees par le minage. Evite la collecte manuelle dans les grands chantiers.
Prospection
Max III Niv. 45+
Chance d'obtenir des drops supplementaires lors du minage de minerais. Distinct de Fortune (probabiliste vs multiplicatif).
I — 15% chance de +1 drop II — 25% chance de +1 drop III — 30% chance de +1 a +2 drops
Compatible avec Fortune vanilla (les effets s'appliquent independamment). Incompatible avec Silk Touch.
Vitesse Miniere
Max III Niv. 35+
Augmente la vitesse de minage, en s'ajoutant a Efficiency. Les deux se cumulent.
I — +10% II — +20% III — +30%
Le bonus est un multiplicateur sur le dig speed block attribute. Implemente via PlayerEvent.BreakSpeed. Se cumule avec Efficiency vanilla (Efficiency 5 + Vitesse III = tres rapide).
Durabilite Cristalline
Max III Niv. 50+
Chance de ne pas consommer de durabilite lors d'une utilisation. Plus puissant qu'Unbreaking vanilla, se cumule avec lui.
I — 15% chance no-durability II — 25% chance III — 35% chance
La probabilite combinee avec Unbreaking V : ~50% chance de ne pas perdre de durabilite (Unbreaking V ~33% + Cristalline III 35% = plafonné a 50% pour equilibre).
Recolte en Masse
Max II Niv. 40+
Casser une culture mature recolte automatiquement toutes les cultures connectees du meme type dans un rayon. Enchantement de houe uniquement.
I — rayon 5 blocs II — rayon 9 blocs
Seules les cultures matures sont recoltees. La propagation suit les blocs connectes en BFS (Breadth-First Search) avec un cap de 512 blocs pour eviter les freezes.
Bucheron
Max II Niv. 45+
Casser un bloc de bois abat automatiquement tout l'arbre entier. Enchantement de hache uniquement.
I — jusqu'a 32 blocs de bois II — jusqu'a 64 blocs de bois

09 Regles d'Incompatibilite

Certains enchantements sont mutuellement exclusifs sur un meme item. Ces incompatibilites sont enforcees au niveau du registre d'enchantements (methode canApplyTogether()) et a la table d'enchantement. L'enclume les autorise si le joueur est pret a payer le cout en niveaux.

Incompatibilites strictes (meme item)

Fleche Explosive + Tir Perforant
Une fleche ne peut pas exploser et percer simultanement. Conflits physiques.
Volee + Infinity (vanilla)
Fleches infinies multipliees = trop puissant. Oblige le choix entre sustain et volume.
Auto-Smelt + Fortune (vanilla) — COMPATIBLE
Synergie autorisee avec trade-off : Fortune s'applique a 50% de rendement quand Auto-Smelt est actif (confort vs quantite max). Silk Touch prend priorite sur les deux.
Absorption d'Impact + Carapace
Deux enchantements defensifs majeurs. La combinaison rendrait le joueur quasi invulnerable aux frappes physiques.
Ancrage II + Onde de Choc II
Les deux modifient le knockback de facon opposee. Ancrage I + Onde de Choc I est autorise.
Represailles + Thorns (vanilla)
Redondance fonctionnelle. Represailles est superieur et les deux sur le meme item doubleraient le reflet.
Fleche Givrante + Fleche Explosive
Les effets de statut ne s'appliquent pas via explosion. Combinaison trompeuse pour le joueur.
Resilience (2 pieces differentes)
Un seul trigger de Resilience actif par joueur. Deux pieces avec Resilience ne se cumulent pas — la plus haute prend priorite.

Compatibilites notables (autorisees et synergiques)

Siphon + Hemorragie
Lifesteal sur hit + saignement = pression soutenue. Les deux sont autorisees car elles ne se chevauchent pas mecaniquement.
Faille + Volatilite
Reduire l'armure progressivement + frapper fort a la 4e/5e frappe = combo de combo. Puissant mais lisible.
Fleche Tracante + Fleche Givrante
Marquer et ralentir la cible = utilitaire d'equipe fort. Synergie intentionnelle.
Excavation + Magnetisme
Miner en grand + attirer les drops = tres efficace. Synergie outils de minage evidente.
Fortification + Carapace
Reduction de degats generaux (immobile) + reduction d'explosion = utile pour les defenseurs de base.
Vitalite + Regeneration Cristalline
Plus de PV max + regen en territoire = survie dans la base de faction. Build defensif coherent.
ℹ
Implementation : Enchantment.canApplyTogether(Enchantment) est override dans chaque classe d'enchantement custom pour retourner false avec ses incompatibles. La validation a l'enclume est aussi verifiee via EnchantmentHelper.areCompatible().

11 Integration RPG & Gemmes

Les enchantements interagissent avec le systeme RPG (stats, perks) et le systeme de Gemmes (accessoires). Cette integration enrichit la profondeur des builds sans rendre les enchantements dependants d'un autre systeme pour fonctionner.

Interaction avec les Gemmes

Les gemmes (accessoires socketables) peuvent synergiser avec certains enchantements. Ces interactions sont documentees dans le spec Accessoires & Gemmes. Resume des synergies cles :

  • Gemme de Siphon + enchantement Siphon → Stack avec le cap global de 25%
  • Gemme de Precision + enchantement Precision (arc) → Le bonus de gravite est multiplie de 1.5x
  • Gemme de Resilience + enchantement Resilience → Reduit le cooldown de 20% supplementaires
  • Gemme de Vitalite + enchantement Vitalite → Les deux s'additionnent, le cap global HP s'applique

Ordre de calcul des degats

Pour eviter les ambiguites d'implementation, l'ordre de calcul est strict :

Degat de base → x Sharpness (vanilla) → x Faille (ignoration armure) → x Stat RPG Armor Penetration → - Armure cible (modifiee par penetration) → - Protection vanilla → - Carapace / Fortification → + Fracture AoE (si critique, calcule independamment) → + Volatilite bonus (si Nth frappe) → Degat final → Hemorragie appliquee (DoT separe) → Siphon soin (sur degat final) → Represailles reflet (sur degat final, envoye a l'attaquant)
⚠
Important : Les caps globaux (Siphon 25%, Vitalite +20 coeurs, Faille 50%) doivent etre verifies dans EnchantEffectHandler avant d'appliquer chaque effet, en cumulant les niveaux de toutes les pieces equipees.

16 Equilibrage & Impact PvP

Philosophie d'equilibrage

Le serveur utilise le PvP 1.8 : pas de cooldown d'attaque, jitter click possible. Les enchantements d'armes ont ete conçus specifiquement pour ce contexte. L'objectif est que chaque enchantement soit attrayant sans etre obligatoire — pas de "meta fixe".

MechaniqueCapRaison

Ignoration armure (Faille + RPG)50%Rend l'armure toujours pertinente meme contre Faille III maxe
Deflexion projectiles40%Les archers restent dangereux
Bonus HP (Vitalite + RPG)+20 coeursEvite les HP pools absurdes
Reduction degats explosion60%La TNT reste une menace meme avec Carapace

Analyse d'impact PvP 1.8 par categorie

Builds synergiques identifies

  • Build Pressuriser : Hemorragie III + Faille III + Siphon II — DoT + penetration progressive + vol de vie. Tres fort en duel prolonge. Counter : Carapace non, mais Resilience + Deflexion oui.
  • Build Soutien : Resonance II + Fracture III — marquer pour les allies et AoE sur critiques. Ideal en combat de groupe faction.
  • Build Defensif Absolu : Vitalite III x4 pieces + Regen Cristalline III x4 — +6 coeurs + regen en territoire. Fort en defense de base mais nul en assaut.
  • Build Archer Controle : Fleche Givrante + Fleche Tracante + Tir Rapide III — ralentir, marquer, tirer vite. Couverture d'equipe excellente.
  • Build Mineur Efficace : Excavation III + Magnetisme II + Auto-Smelt + Bucheron II — miner une montagne en quelques minutes.
⚠
Surveillance post-launch : Les valeurs numeriques (pourcentages, durees, cooldowns) sont toutes dans EnchantConfig et modifiables sans recompilation. Une periode de test de 2 semaines est prevue avant de figer les valeurs de production.

  • Chaque enchantement : trigger au bon moment (hit, tick, kill, etc.)

  • Incompatibilites : impossible d'appliquer deux enchantements incompatibles via table
  • Caps globaux : lifesteal ne depasse pas 25% meme avec Siphon III + stat RPG max
  • Echo Defensif : cooldown respecte, 3 frappes en 2s = trigger, 3 frappes en 3s = pas de trigger
  • Cristallisation : 3 kills consecutifs = absorption, mort remet le compteur a 0
  • Bucheron : Sneak + hache = minage individuel, sans Sneak = arbre entier

Tests de combat PvP (scenarios 1v1)

  • Scenario A — Pressuriser vs Defensif : Hemorragie III + Faille III + Siphon II vs Resilience II + Carapace III + Vitalite III. Duree attendue : 15-25 secondes (ni trop court ni trop long).

  • Scenario C — Build vanilla vs Build Erinium : Sharpness V + Protection IV vanilla vs Sharpness VII + Siphon III + Faille II. Le build Erinium doit etre clairement superieur mais pas de l'ordre du one-shot.
  • Scenario D — Combat de groupe 3v3 : Resonance II actif sur les deux equipes. Verifier que les bonus de degats allies s'appliquent correctement aux bonnes cibles.

  • Excavation 3x3 x 10 joueurs simultanement : server TPS reste > 18
