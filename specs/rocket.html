Fusee & Rocket Maker

Phase 5 de la Spatial Update. Systeme complet de fusee intere-dimensionnelle pour transporter les joueurs entre l'Overworld et Erina (DIM 42).

1. Contexte & Objectifs

La Fusee est le seul moyen legitime d'acceder a la dimension Erina. Elle represente un investissement lourd de la faction : craft cher, 5 composants a assembler, et risque de destruction PvP sur Erina. C'est une mecanique strategique cle de la Spatial Update : la faction qui controle les fusees controle les expeditions.

  • Craft Bloc Rocket Maker (patron ENE/DRD/EBE) sert d'atelier d'assemblage
  • Composants 5 types a crafter separement (Coque x4, Reacteur, Reservoir, Navigation, Siege x2)
  • Entite EntityRocket placee dans le monde, 100 HP, 2 passagers max
  • Destruction PvP La fusee peut etre abattue sur Erina, bloquant l'expedition

2. Composants de la Fusee

L'assemblage de la fusee necessite exactement 9 items repartis dans les 9 slots du Rocket Maker :

Resultat : 1 item_rocket (stack=1) qui peut etre pose dans le monde pour faire apparaitre l'EntityRocket.

3. Rocket Maker — Bloc d'assemblage

Bloc special dedie a l'assemblage de la fusee. N'est pas une crafting table : il a son propre GUI multi-slots.

Craft du bloc

Patron shaped (3x3) : E N E D R D E B E E = erinium_fragment (EriniumFaction) N = nether_star D = diamond_block R = redstone_block B = blaze_rod

Classes Java

  • BlockRocketMaker (erina/block/) — extends Block implements ITileEntityProvider. PropertyDirection FACING (4 directions horizontales). hasTileEntity=true. onBlockActivated ouvre le GUI via player.openGui(EriniumFaction.instance, GUI_ID, world, x, y, z).
  • TileEntityRocketMaker (erina/tile/) — extends TileEntity implements ITickable. Stocke 9 slots via ItemStackHandler, un champ int assemblyProgress (0-200) et boolean assemblyActive. Validation par slot : chaque slot accepte uniquement son composant attendu.
  • ContainerRocketMaker (erina/container/) — 9 slots TE + 36 slots inventaire joueur. SlotRocketComponent extends SlotItemHandler refuse les mauvais ingredients (retour false dans isItemValid).
  • GuiRocketMaker (erina/gui/) — extends GuiContainer. Affiche les 9 slots avec bordures vertes/rouges selon validation, bouton "Assembler" qui envoie un custom payload, barre de progression 0-200 ticks.

Validation par slot

TileEntityRocketMaker.isValidForSlot(int slot, ItemStack stack) compare le registry name du stack au composant attendu. Table :

Slot indexComposant attendu
0-3eriniumfaction:rocket_hull
4eriniumfaction:rocket_engine
5eriniumfaction:rocket_tank
6eriniumfaction:rocket_nav
7-8eriniumfaction:rocket_seat

GUI

Inspiree du style sobre des anciens GuiContainer. 9 slots en layout 3x3 au centre, labels au-dessus de chaque slot. Bordure verte si le slot contient le bon ingredient, rouge si mauvais, neutre si vide. Bouton "Assembler" en bas, active uniquement quand les 9 slots sont valides. Barre de progression visible pendant les 200 ticks d'assemblage. A la fin, les 9 slots sont consommes et item_rocket est place dans le dernier slot vide du joueur (ou dropped au sol).

4. EntityRocket — entite placeable

Classe

EntityRocket (erina/entity/) extends Entity. Enregistree dans EriniumFaction.preInit via EntityRegistry.registerModEntity(new ResourceLocation(MODID,"rocket"), EntityRocket.class, "rocket", 71, this, 80, 3, true).

NBT

{ "FuelCount": 1-4, // nombre de Blaze Rods restants "Health": 0-100, // HP actuels "OwnerUUID": "uuid", // proprietaire (celui qui a pose) "FactionId": "id", // faction du proprio (optionnel) "OriginX": int, // X de depart en Overworld "OriginY": int, // Y de depart "OriginZ": int, // Z de depart "OriginDim": int // dimension de depart (normalement 0) }

Embarquement

  • Clic droit sur l'entite = startRiding(rocket). Capacite 2 passagers (donnee par canFitPassenger).
  • Si deja 2 passagers : message erreur au 3eme qui tente d'embarquer.
  • updatePassenger replace le passager juste au-dessus du siege (offset +0.8 Y par passager, X/Z identiques a l'entite).

Destruction PvP

  • attackEntityFrom(DamageSource src, float amount) decremente health. Si 0 : setDead(), ejection des passagers a la position actuelle, drop d'un peu de feraille (optionnel).
  • La fusee ne regenere pas.
  • canBeCollidedWith = true, canBePushed = false.
Destruction en vol : Si la fusee est detruite sur Erina alors qu'un joueur est dedans, le passager est ejecte a la position de l'entite (pas teleporte ailleurs). Il reste coince sur Erina jusqu'a une autre fusee ou sa mort.

5. Sequence de decollage

Declenchement

Le joueur pilote (premier embarque) appuie sur la touche R (configurable via EriKeys, contexte IN_GAME). Le client envoie un RocketLaunchPacket au serveur avec l'ID de l'entite fusee.

Checks cote serveur

  1. Le joueur est bien passager pilote de l'entite ?
  2. L'entite est bien une EntityRocket ?
  3. Le joueur a au moins 1 Blaze Rod dans son inventaire ?
  4. Un decollage n'est pas deja en cours (flag launching) ?

Si un check echoue : message erreur chat, aucune action. Si tous OK : consomme 1 Blaze Rod, active le flag launching, envoie un RocketLaunchStartPacket a tous les passagers.

Sequence visuelle (20 + 60 = 80 ticks)

  • Ticks 0-20 (tremblement) : particules de feu sous la fusee (FLAME, 20 par tick), son item.rocket.launch joue une fois au tick 0, camera du passager secoue (player.rotationYaw += sin(t*0.5)*1.5).
  • Ticks 20-80 (ecran noir) : overlay noir qui fade-in sur 60 ticks (alpha 0 -> 255). Aucune action possible pour le passager.
  • Tick 80 (teleportation) : tous les passagers sont teleportes. L'ecran reste noir 20 ticks supplementaires cote client puis fade-out.

Destination

  • Overworld → Erina : spawn fixe X=0, Z=0, Y=surface calculee via findSafeSpawn. La fusee reste sur place en Overworld (les passagers sont ejectes AVANT la teleportation).
  • Erina → Overworld : spawn aux coordonnees OriginX/OriginY/OriginZ de la fusee. Si OriginDim est invalide, fallback au spawn Overworld.
  • Utilise TeleportHelper.teleport(player, x, y, z, yaw, pitch, dimId) (systeme existant).

Consommation carburant

1 Blaze Rod est consomme dans l'inventaire du pilote a l'instant T=0 de la sequence. Si le vol est interrompu (rocket detruite en vol), le Blaze Rod n'est pas rembourse.

6. ItemRocket

ItemRocket (erina/item/) extends Item. setMaxStackSize(1). onItemUse : place un EntityRocket a la position cliquee, copie le NBT de l'ItemStack vers l'entite (OwnerUUID = player.uuid, OriginX/Y/Z = pos cliquee, OriginDim = player.dimension, Health = 100, FuelCount = 0). L'item est consomme.

7. Integration ClientProxy

  • RenderingRegistry.registerEntityRenderingHandler(EntityRocket.class, RenderRocket::new) (rendu simple cube colore en v1, modele Blockbench en v2)
  • EriKeys.create("eriniumfaction.rocket.launch").key(Keyboard.KEY_R).context(KeyContext.IN_GAME).onPress(() -> RocketLaunchHandler.tryLaunchFromClient())
  • GuiNetworkHandler enregistre l'action rocket_maker_assemble cote serveur
  • OverlayModManager.register(new RocketLaunchOverlay()) pour l'ecran noir de decollage
