Accessoires & Gemmes

Systeme d'accessoires via Baubles API (anneaux, amulettes, bonus de set) et systeme de gemmes avec sertissage dans l'equipement et les accessoires.

A faire Baubles API Valterite — Tier 2 Aetherite — Tier 3 Erinium — Tier 4 6 types de gemmes 3 qualites

01 Contexte & Objectifs

EriniumFaction ajoute deux systemes complementaires qui enrichissent la profondeur de la progression PvP : les Accessoires (Baubles API) et les Gemmes & Sertissage. Ces deux systemes sont fortement integres l'un dans l'autre et dans le systeme RPG existant.

Pourquoi ces systemes ?

  • Accessoires — Permettre aux joueurs d'equiper des items dans des slots dedies (anneaux, amulette) hors de l'inventaire vanilla, avec des bonus passifs et des synergies de set entre tiers.
  • Gemmes — Ajouter une couche de personnalisation en encastrant des gemmes dans l'equipement. Chaque gemme apporte un bonus de stat RPG ET un effet de combat distinct.
  • Sertissage — Systeme de sockets par item (0 a 3 selon le tier) avec une interface GUI dediee, gerant l'insertion et l'extraction des gemmes.
  • Integration RPG — Tous les bonus (accessoires + gemmes) sont visibles et cumules dans le panneau RPG du joueur, maintenant une source de verite unique sur les stats.

ℹ
Lecture prerequise — Avant d'implementer ce systeme, lire docs/specs/rpg-system.html (stats RPG, RpgPlayerData, RpgEventHandler) et docs/specs/combat-equipment.html (tiers Valterite/Aetherite/Erinium, lingots, materiaux). Les deux sont des dependances directes de ce cahier des charges.
⚠
PvP 1.8 — ATTACK_SPEED inutile — L'attribut ATTACK_SPEED est ignore sur tout le projet. Les effets de combat des gemmes et accessoires concernent uniquement : les degats bruts, le knockback, la vitesse de mouvement, le lifesteal, et les chances de drop. Ne jamais coder de mecanique basee sur ATTACK_SPEED.

SetBonusManager — logique de verification

public class SetBonusManager { private static final SetBonusManager INSTANCE = new SetBonusManager(); public static SetBonusManager getInstance() { return INSTANCE; } public void checkAndApplySetBonus(EntityPlayerMP player) { IItemHandler baubles = BaublesApi.getBaublesHandler(player); for (AccessoryTier tier : AccessoryTier.values()) { boolean hasRing = hasBaubleOfTier(baubles, tier, BaubleType.RING); boolean hasAmulet = hasBaubleOfTier(baubles, tier, BaubleType.AMULET); if (hasRing && hasAmulet) { applySetBonus(player, tier); } } } public void checkAndRemoveSetBonus(EntityPlayerMP player) { IItemHandler baubles = BaublesApi.getBaublesHandler(player); for (AccessoryTier tier : AccessoryTier.values()) { boolean hasRing = hasBaubleOfTier(baubles, tier, BaubleType.RING); boolean hasAmulet = hasBaubleOfTier(baubles, tier, BaubleType.AMULET); if (!hasRing || !hasAmulet) { removeSetBonus(player, tier); } } } private boolean hasBaubleOfTier(IItemHandler baubles, AccessoryTier tier, BaubleType type) { for (int i = 0; i < baubles.getSlots(); i++) { ItemStack stack = baubles.getStackInSlot(i); if (!stack.isEmpty() && stack.getItem() instanceof BaseAccessoryItem) { BaseAccessoryItem item = (BaseAccessoryItem) stack.getItem(); if (item.getTier() == tier && item.getBaubleType(stack) == type) return true; } } return false; } private void applySetBonus(EntityPlayerMP player, AccessoryTier tier) { RpgPlayerData data = DataManager.get(player, RpgPlayerData.class); if (!data.activeSetBonuses.contains(tier.name())) { data.activeSetBonuses.add(tier.name()); // Appliquer le bonus specifique du tier tier.getSetBonus().apply(player, data); DataManager.save(player, RpgPlayerData.class); } } }
⚠
Double anneau et set bonus — Deux anneaux du meme tier NE declenchent PAS le bonus de set. Il faut obligatoirement anneau + amulette du meme tier. Deux anneaux de tiers differents cumulent simplement leurs stats individuelles.

Gemme droppee : selection aleatoire

Quand un drop de gemme est declenche, le type de gemme est choisi aleatoirement parmi les 6 types avec probabilites egales (1/6 chacun). La qualite est toujours BRUTE pour les drops.

// Implementation dans RpgEventHandler (listener sur HarvestDropsEvent) @SubscribeEvent public void onHarvestDrops(BlockEvent.HarvestDropsEvent event) { if (event.getHarvester() == null) return; // Pas de joueur EntityPlayerMP player = (EntityPlayerMP) event.getHarvester(); Block block = event.getState().getBlock(); float baseChance = GemDropTable.getChance(block); if (baseChance <= 0) return; int fortuneLevel = EnchantmentHelper.getEnchantmentLevel( Enchantments.FORTUNE, player.getHeldItemMainhand()); RpgPlayerData data = DataManager.get(player, RpgPlayerData.class); float finalChance = Math.min(50f, baseChance + (fortuneLevel * 2f) + (data.fortune * 0.5f)); if (player.getRNG().nextFloat() * 100f < finalChance) { GemType randomType = GemType.values()[player.getRNG().nextInt(GemType.values().length)]; GemItem gemItem = GemItems.getItem(randomType, GemQuality.RAW); event.getDrops().add(new ItemStack(gemItem)); } }
⚠
Silk Touch — Avec Silk Touch, aucune gemme n'est droppee (le bloc est recupre entier). Verifier event.isSilkTouching() avant de calculer le drop.

Structure NBT d'un socket — SocketData

Les donnees de socket sont stockees directement dans le NBT de l'ItemStack sous la cle "SocketData". Chaque socket est une entree dans un NBTTagList.

10 Sertissage — Insertion & Extraction

Insertion d'une gemme

L'insertion se fait via le GUI de sertissage (Section 11). Cliquer droit sur un enclume avec un item socketable dans la main ouvre l'interface. Le joueur peut alors faire glisser une gemme de son inventaire vers un socket vide et confirmer.

  • Prerequis — L'item doit avoir au moins un socket vide (verifie par SocketManager)
  • Action — La gemme (item) est retiree de l'inventaire du joueur et ses donnees sont ecrites dans le NBT du socket cible
  • Irreversible — Sans Extracteur d'Erinium, retirer une gemme la detruit
  • Feedback — Son de placement + particule de la couleur de la gemme au moment de la confirmation

Extraction : gemme detruite (par defaut)

Retirer une gemme via le GUI sans Extracteur la detruit. Le socket redevient vide, mais la gemme est perdue. Un avertissement est affiche avant confirmation.

⚠
Extraction sans Extracteur = destruction — C'est une decision de design intentionnelle pour donner de la valeur aux gemmes et a l'Extracteur d'Erinium. L'interface GUI doit clairement afficher cet avertissement avant de permettre la suppression.
