More ModOverlays

Collection de 12 overlays HUD indépendants et configurables : FPS, ping, TPS, keystrokes, heure, coordonnées, boussole, biome, armure, potions et plus. Chacun utilise le système OverlayMod d'EriAPI avec un moteur de presets graphiques partagé.

01Contexte & Objectifs

Regles fondamentales

!
JAMAIS d'overlay brut : Conformément aux règles du projet, tous les overlays DOIVENT étendre OverlayMod d'EriAPI et être enregistrés via OverlayModManager. N'utiliser ni RenderGameOverlayEvent direct, ni OpenGL brut pour le rendu HUD.

Objectifs

  • 12 overlays indépendants — chacun activable/désactivable séparément par le joueur
  • Système de presets — 6 styles graphiques prédéfinis + Custom avec color pickers
  • Preset RGB — animation arc-en-ciel continue via AnimationManager avec timer global (pas un timer par overlay)
  • Coordonnées design 1920x1080 — le ScaleManager d'EriAPI gère la conversion vers la résolution réelle
  • Performance — aucune allocation dans le hot path de rendu

02Liste des 12 Overlays

OVERLAY 01
FPS Counter
FpsOverlay
Affiche les FPS actuels du client. Couleur dynamique selon performance : vert si >60, jaune si >30, rouge si <30.
  • showLabel — afficher "FPS" ou juste le nombre
  • textColor — couleur fixe (ignorée si preset actif)
  • scale — multiplicateur de taille (0.5–2.0)
  • dynamicColor — activer la coloration selon les seuils
OVERLAY 02
Ping Display
PingOverlay
Latence en millisecondes via player.connection.getPlayerInfo(uuid).getResponseTime(). Couleur dynamique : vert <50ms, jaune <100ms, rouge >100ms.
  • showLabel — afficher "ms" après le nombre
  • dynamicColor — coloration selon les seuils de latence
  • scale — multiplicateur de taille
OVERLAY 03
Server TPS
TpsOverlay
TPS du serveur synchronisé depuis le serveur via packet custom. Couleur : vert >18, jaune >15, rouge <15. Affiche "?" si aucune donnée reçue.
  • showLabel — afficher "TPS"
  • dynamicColor — coloration selon les seuils
  • decimals — 0 ou 1 décimale
OVERLAY 04
Keystroke Display
KeystrokeOverlay
Affiche visuellement les touches WASD + Espace + Shift + Ctrl + LMB + RMB. Animation pressed/released. CPS intégré dans LMB/RMB.
  • showCps — afficher le CPS dans LMB/RMB
  • colorPressed — fond quand touche enfoncée
  • colorReleased — fond quand touche relâchée
  • keySize — taille des touches en px design
OVERLAY 05
In-Game Time
GameTimeOverlay
Heure du monde Minecraft (0-24h). Icône soleil ☀ de 6h à 18h, lune ☾ de 18h à 6h. Conversion via world.getWorldTime() % 24000.
  • format24h — format 24h ou 12h (AM/PM)
  • showIcon — afficher l'icône soleil/lune
  • showDayCount — afficher le jour (Jour 5)
OVERLAY 06
Real Time Clock
RealTimeOverlay
Heure réelle du PC via Calendar.getInstance(). Format configurable. Mise à jour chaque tick client.
  • format24h — format 24h ou 12h
  • showSeconds — afficher les secondes (HH:MM:SS)
  • textColor — couleur du texte
OVERLAY 07
Compass
CompassOverlay
Boussole horizontale avec barre graduée. La direction courante du joueur est centrée. Affiche N/S/E/W et les degrés. Largeur configurable.
  • width — largeur de la barre (200–600 px design)
  • colorCardinal — couleur des lettres cardinales
  • colorTick — couleur des graduations
  • showDegrees — afficher les degrés en dessous
OVERLAY 08
Coordinates XYZ
CoordsOverlay
Position XYZ du joueur via player.posX/Y/Z. Chaque axe configurable individuellement. Supporte 0 ou 2 décimales.
  • showX/Y/Z — afficher chaque axe
  • decimals — 0 ou 2 décimales
  • orientation — horizontal ou vertical
  • colorX/Y/Z — couleur par axe
OVERLAY 09
Biome Display
BiomeOverlay
Nom du biome sous le joueur. Utilise le registre EriniumBiomeProvider pour le nom custom. Via world.getBiome(player.getPosition()).getBiomeName().
  • showTemperature — afficher la température du biome
  • textColor — couleur du texte
OVERLAY 10
Direction / Facing
FacingOverlay
Direction cardinale du joueur via player.getHorizontalFacing(). Format long ("North +Z") ou court ("N"). Converti depuis les degrés de rotation yaw.
  • longFormat — "North (+Z)" vs "N"
  • showAxis — afficher l'axe Z/X
  • textColor — couleur
OVERLAY 11
Armor Status
ArmorOverlay
Durabilité des 4 pièces d'armure avec barres colorées. Rouge si <20%, jaune si <50%, vert sinon. Affiche optionnellement les icônes d'items.
  • orientation — vertical ou horizontal
  • showItemIcons — afficher les icônes d'armure
  • showPercentage — afficher le % de durabilité
  • hideEmpty — masquer les slots vides
OVERLAY 12
Potion Effects
PotionOverlay
Liste des effets de potion actifs via player.getActivePotionEffects(). Affiche le nom, le niveau et la durée restante. Icônes optionnelles.
  • showIcons — afficher les icônes de potions
  • showAmplifier — afficher le niveau (II, III)
  • showDuration — afficher le temps restant
  • maxEffects — nombre max à afficher (1–10)

03Systeme de Presets Graphiques

Les 6 presets

Default
Fond noir translucide 0x80000000, texte blanc 0xFFFFFFFF, bordure grise 0x40FFFFFF. Style discret universel.
Liquid Glass
Fond 0xA0101020, bordure 0x20FFFFFF, texte 0xFFE8E8F0. Style cohérent avec les GUIs du mod.
Neon
Fond noir 0xCC000000, texte cyan 0xFF00E5FF, bordure 0x40FFFFFF. Style cyberpunk avec lueur sur le texte.
RGB
Couleurs défilent en arc-en-ciel. Timer global partagé. Hue +1/tick. Appliqué au texte et/ou à la bordure.
Minimal
Pas de fond, pas de bordure. Texte seul avec ombre portée. Style épuré pour les puristes.
Custom
Color pickers complets. L'utilisateur choisit fond, texte, bordure, accent individuellement via ColorPicker EriAPI.

Structure OverlayStylePreset

public enum OverlayStylePreset { DEFAULT( 0x80000000, // bgColor 0xFFFFFFFF, // textColor 0x40FFFFFF, // borderColor 0xFF00E5FF // accentColor ), LIQUID_GLASS(0xA0101020, 0xFFE8E8F0, 0x20FFFFFF, 0xFF9B4FCF), NEON (0xCC000000, 0xFF00E5FF, 0x60FFFFFF, 0xFF00E5FF), RGB (0xCC000000, 0xFFFFFFFF, 0x80FFFFFF, 0xFFFFFFFF), // couleurs overridées par le timer MINIMAL (0x00000000, 0xFFFFFFFF, 0x00000000, 0xFFAAAAAA), CUSTOM (0x80000000, 0xFFFFFFFF, 0x40FFFFFF, 0xFF00E5FF); // valeurs par défaut public final int bgColor, textColor, borderColor, accentColor; OverlayStylePreset(int bg, int text, int border, int accent) { ... } }

OverlayStyleManager — singleton

public class OverlayStyleManager { private static final OverlayStyleManager INSTANCE = new OverlayStyleManager(); private OverlayStylePreset activePreset = OverlayStylePreset.LIQUID_GLASS; private float globalOpacity = 0.85f; // slider 0–100% dans l'éditeur private float globalScale = 1.0f; // slider 0.5–2.0x // Couleurs custom (utilisées si preset == CUSTOM) private int customBg, customText, customBorder, customAccent; // Couleur RGB courante (mise à jour par le timer global) private int rgbColor = 0xFFFF0000; public int getTextColor() { if (activePreset == OverlayStylePreset.RGB) return rgbColor; if (activePreset == OverlayStylePreset.CUSTOM) return customText; return activePreset.textColor; } public int getBgColor() { int base = (activePreset == OverlayStylePreset.CUSTOM) ? customBg : activePreset.bgColor; // Appliquer globalOpacity sur l'alpha int alpha = (int) (((base >> 24) & 0xFF) * globalOpacity); return (alpha << 24) | (base & 0x00FFFFFF); } }

04Preset RGB — Animation Arc-en-Ciel

Timer global unique

Un seul timer RGB pour tous les overlays actifs en preset RGB. Il tourne de 0 à 360 (hue HSL) sur environ 5 secondes (300 ticks), puis boucle en infini. La couleur calculée est stockée dans OverlayStyleManager.rgbColor et lue par tous les overlays.

// Initialisation du timer RGB dans OverlayStyleManager.init() private float hue = 0f; public void init() { // Animation loop infinie — hue 0→360 en 300 ticks (~15s à 20tps) Animation.create(0f, 360f, 300) .loop() .easing(Easing.LINEAR) .onUpdate(h -> { hue = h; rgbColor = hslToArgb(h, 1.0f, 0.65f); }); AnimationManager.getInstance().play(rgbAnimation); } // Conversion HSL → ARGB (alpha = 0xFF) private int hslToArgb(float h, float s, float l) { // Formule standard HSL → RGB h = h % 360f; float c = (1f - Math.abs(2f * l - 1f)) * s; float x = c * (1f - Math.abs((h / 60f) % 2f - 1f)); float m = l - c / 2f; // ... calcul r, g, b selon la plage de h ... return 0xFF000000 | (r << 16) | (g << 8) | b; }

Application aux elements

En preset RGB, le timer est appliqué au texte et à la bordure. L'application sur le fond est configurable (par défaut désactivé pour la lisibilité). Chaque overlay récupère OverlayStyleManager.getInstance().getTextColor() à chaque frame.

✓
Performance : Un seul objet Animation tourne pour tous les overlays RGB. Le coût est O(1) par frame, peu importe le nombre d'overlays actifs en mode RGB.

05TPS Packet Reseau

Probleme

Le TPS du serveur n'est pas accessible nativement côté client. Il faut créer un packet custom pour synchroniser cette valeur périodiquement.

Solution — Packet TPS

// Packet S→C envoyé toutes les 20 ticks (1 seconde) public class PacketTpsUpdate implements IMessage { public float tps; // TPS calculé côté serveur // Calcul côté serveur : tickTime = server.recentTps[0] // TPS = Math.min(20.0, 1000.0 / server.tickTimeArray[i]) // Disponible via MinecraftServer.getServer().recentTps } // Envoi depuis un scheduler serveur (toutes les 20 ticks) EriScheduler.repeat(20, () -> { float tps = (float) Math.min(20.0, MinecraftServer.getServer().recentTps[0]); for (EntityPlayerMP player : server.getPlayerList().getPlayers()) { EriniumFaction.CHANNEL.sendTo(new PacketTpsUpdate(tps), player); } }); // Côté client — stockage dans TpsOverlay public class PacketTpsUpdateHandler implements IMessageHandler<PacketTpsUpdate, IMessage> { public IMessage onMessage(PacketTpsUpdate msg, MessageContext ctx) { Minecraft.getMinecraft().addScheduledTask(() -> TpsOverlay.getInstance().setCurrentTps(msg.tps) ); return null; } }

06Keystroke Display — Detail

Layout des touches

Le layout reproduit un clavier physique en grille 3x4. Les touches sont des rectangles avec label centré.

// Layout en coordonnées design (pixels 1920x1080 relatifs au coin de l'overlay) // Taille de chaque touche : 40x40 px design (configurable via keySize) // Espacement : 4 px entre touches // Rangée 1 : [ ] [W] [ ] ← (vide) W (vide) // Rangée 2 : [A] [S] [D] ← A S D // Rangée 3 : [CTRL] [SPACE] [SHIFT] // Rangée 4 : [ LMB ] [ RMB ] ← avec CPS

Detection des touches

// Détection des touches WASD (keybindings vanilla) Minecraft mc = Minecraft.getMinecraft(); boolean wPressed = mc.gameSettings.keyBindForward.isKeyDown(); boolean aPressed = mc.gameSettings.keyBindLeft.isKeyDown(); boolean sPressed = mc.gameSettings.keyBindBack.isKeyDown(); boolean dPressed = mc.gameSettings.keyBindRight.isKeyDown(); boolean spPressed = mc.gameSettings.keyBindJump.isKeyDown(); boolean shPressed = mc.gameSettings.keyBindSneak.isKeyDown(); // LMB / RMB via Mouse.isButtonDown() boolean lmbDown = Mouse.isButtonDown(0); boolean rmbDown = Mouse.isButtonDown(1);

Calcul CPS (clics par seconde)

// Buffer circulaire des timestamps de clics — taille fixe de 100 private final long[] lmbClickTimes = new long[100]; private int lmbClickHead = 0; private int lmbClickCount = 0; // À chaque nouveau clic LMB (front montant) void onLmbClick() { lmbClickTimes[lmbClickHead % 100] = System.currentTimeMillis(); lmbClickHead++; if (lmbClickCount < 100) lmbClickCount++; } // Calcul CPS — compte les clics dans la dernière seconde int getCps() { long now = System.currentTimeMillis(); int cps = 0; for (int i = 0; i < lmbClickCount; i++) { if (now - lmbClickTimes[i] <= 1000L) cps++; } return cps; }
ℹ
Pas d'ATTACK_SPEED : Conformément aux règles du projet, le serveur utilise le PvP 1.8 sans cooldown d'attaque. Le CPS affiché est donc pertinent et utile pour les joueurs qui font du jitter click.

10Performance

Contexte client

Les overlays tournent côté client uniquement et s'exécutent chaque frame dans le thread de rendu. Les contraintes de performance sont différentes du serveur mais restent importantes pour garder un framerate fluide.

  • Pas d'allocation dans render() — les String de texte sont formatés en amont, pas à chaque frame. Utiliser des champs String pré-calculés mis à jour à une fréquence réduite
  • Mise à jour découplée du rendu — FPS, coordonnées, biome : calculé chaque tick (20x/sec), pas chaque frame. Le rendu lit la valeur pré-calculée
  • Timer RGB unique — un seul AnimationManager.play() pour tous les overlays RGB, pas un timer par overlay actif
  • CPS buffer tableau primitif — long[] fixe, pas de List ni d'ArrayList dynamique
  • Compass — les positions des markers cardinaux sont recalculées seulement si le yaw a changé de plus de 0.5 degrés
  • Biome — recalculé uniquement si le joueur a bougé de plus d'un chunk depuis la dernière lecture
// Pattern "tick-cached" — exemple CoordsOverlay private String cachedText = ""; private int tickTimer = 0; @Override public void onTick(EntityPlayer player) { if (tickTimer++ % 4 != 0) return; // Met à jour 5x/sec cachedText = String.format("X: %.0f Y: %.0f Z: %.0f", player.posX, player.posY, player.posZ); } @Override public void render(int w, int h) { drawString(cachedText, ...); // Pas de calcul ici }
